# ISO 27001:2022 ISMS Module

## Overview

The ISO 27001:2022 module in IdaraOS helps organizations establish, implement, maintain, and continually improve an **Information Security Management System (ISMS)**. It maps directly to the standard's mandatory clauses (4–10) and Annex A controls (93 controls across four themes).

This module is part of the Security (GRC) area and integrates with the generic foundation layer: risks, controls, evidence, audits, and objectives.

## ISO 27001:2022 Structure

| Section | IdaraOS Coverage |
|---------|------------------|
| **Clauses 4–10** (mandatory ISMS requirements) | ISMS Clauses tracker — compliance status, owners, evidence |
| **Clause 6.2** (security objectives) | Security Objectives — periods, achievement status, evidence links |
| **Annex A** (93 controls) | SoA, Annex A browser, gap analysis, control mappings |
| **Evidence** | Evidence Store + per-control links + objective/clause evidence |

## Sub-Modules

### Dashboard (`/security/frameworks/iso-27001`)

Central view of ISMS compliance: Annex A category progress, clause summary, certification details, and quick links to all sub-modules.

### ISMS Clauses (`/security/frameworks/iso-27001/clauses`)

Tracks mandatory management system requirements from ISO 27001:2022 Clauses 4–10:

- **Clause 4** — Context of the organization
- **Clause 5** — Leadership
- **Clause 6** — Planning (includes risk assessment and **security objectives**)
- **Clause 7** — Support
- **Clause 8** — Operation
- **Clause 9** — Performance evaluation
- **Clause 10** — Improvement

Each requirement records compliance status, owner, implementation notes, and linked evidence.

### Security Objectives (`/security/frameworks/iso-27001/objectives`)

Implements **Clause 6.2 — Information security objectives and planning to achieve them**.

Objectives must be:

- Consistent with the information security policy
- Measurable (or capable of evaluation)
- Monitored and communicated
- Updated as appropriate
- Documented as **retained information**

IdaraOS tracks:

| Field | Purpose |
|-------|---------|
| Reporting period | e.g., FY 2026, Q1 2026 |
| Period start / end | Date boundaries for measurement |
| Achievement status | Not Measured, Not Achieved, Partially Achieved, Achieved |
| Success criteria | Measurable targets (ISO 6.2) |
| Linked evidence | Measurement records, progress reports, resource plans |

### Statement of Applicability (`/security/frameworks/iso-27001/soa`)

Documents which of the 93 Annex A controls are applicable, implemented, or excluded — with justification for exclusions.

### Annex A Controls (`/security/frameworks/iso-27001/controls`)

Browse the pre-loaded ISO 27001:2022 Annex A control library and create organization controls from standard references.

### Evidence Matrix (`/security/frameworks/iso-27001/evidence`)

Matrix view of evidence coverage per SoA item and control implementation.

### Gap Analysis (`/security/frameworks/iso-27001/gaps`)

Identifies controls not yet implemented and prioritizes remediation.

## Evidence Linking

Evidence supports audit and certification readiness through three patterns:

1. **Control ↔ Evidence** — junction table (`security_evidence_links`) for Annex A controls
2. **Clause compliance** — `linkedEvidenceIds` on ISMS clause records
3. **Security objectives** — `linkedEvidenceIds` on objectives for Clause 6.2 measurement records
